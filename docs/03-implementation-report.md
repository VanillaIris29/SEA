
# 03 – Implementation Report

## 1. Реализирана функционалност

В този етап от разработката беше реализиран цялостен функционален модул за управление на курсове, включващ:

### ✔ CRUD функционалност за курсове
- Създаване на курс (Create)
- Редактиране на курс (Edit)
- Изтриване на курс (Delete)
- Преглед на детайли (Details)

### ✔ Управление на участници
- Преглед на записани участници в курс
- Преглед на детайли за конкретен участник
- Ограничение за достъп (само организаторът или администраторът)

### ✔ Търсене, филтриране и сортиране
- Търсене по заглавие и категория
- Филтриране по сертификат
- Сортиране по цена, продължителност, име
- Странициране (pagination)

### ✔ Управление на роли
- Различни изгледи за Admin, Organizer и User
- Ограничения чрез `[Authorize(Roles = "...")]`

### ✔ Качване на изображения
- Записване на снимка за курс
- Генериране на уникално име на файла
- Съхранение в `wwwroot/images/courses`

## 2. Как работи

### Основен поток на работа:

1. **Организаторът влиза в системата** чрез Identity.
2. Отива в **Course → Manage**, където вижда само своите курсове.
3. Може да:
   - създаде нов курс;
   - редактира съществуващ;
   - изтрие курс;
   - види участниците.
4. Потребителите виждат всички активни курсове в **UserCourses → All**.
5. При избор на курс се отваря **Details**, където:
   - ако има места → може да се запише;
   - ако е пълен → се показва съобщение.
6. Администраторът има достъп до **ManageAll**, където управлява всички курсове.
7. Има отделни изгледи за **изтекли курсове** (ManageAllExpired / ManageExpired).

## 3. Възникнали проблеми

### ❗ Проблем 1: 404 грешка при Participants страница
При опит за отваряне на `/Course/Participants/{id}` се получаваше 404.

### ❗ Проблем 2: Невалидни съобщения при логин бяха на английски
Identity връщаше:
- “Invalid login attempt”
- “The Email field is not a valid e-mail address”

### ❗ Проблем 3: Pagination губеше параметрите при филтриране
При преминаване към следваща страница филтрите се нулираха.

---

## 4. Как бяха решени

### ✔ Решение на Проблем 1
Добавен е route атрибут:

[HttpGet("Course/Participants/{id}")]
### ✔ Решение на Проблем 2
[Required(ErrorMessage = "Имейлът е задължителен.")]
[EmailAddress(ErrorMessage = "Моля, въведете валиден имейл адрес.")]

✔ Решение на Проблем 3
Добавени са ViewBag параметри за запазване на филтрите:

csharp
ViewBag.Search = search;
ViewBag.CategoryId = categoryId;
ViewBag.Sort = sort;
ViewBag.Certificate = certificate;
